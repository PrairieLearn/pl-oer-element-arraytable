These questions help you understand how data-fixed-width, strict-grading, and partial-credit affect how answers are shown and graded.

Part 1

If you want to make sure students input the correct number of digits (especially for binary or hex values), you can set data-fixed-width to the number of digits the answer should be (after the prefix). If strict-grading is set to true, then they will be marked as incorrect if the number of digits does not match. If it's set to false, the answer will be invalid and the student won't be able to submit it.

In the first question, data-fixed-width is set to 3 and strict-grading is set to false. (unknown-value is set to "unknown" and will not be considered when checking the width.)
