These questions demonstrate how the index and address features work using array inputs. You'll perform simple additions at each indexed address.

Part 1 - Understanding Index

In this case we set index to be 5 and index-prefix to be"$". Here we can see that the entire index row is filled with values starting from 5, incremented by 1 and prefixed with $. This shows that when we pass in a single value to the index all the other values are filled in by incrementing them by 1 and when we specify the prefix it adds that as well. The array starts at index $5. Fill in each value with the sum of its address and 1.
For example, at $5, enter "6"; at $6, enter "7", and so on.

Part 2 - Empty Index Prefix

We can also pass in an empty string to the index prefix to render the index without any prefix. In this case we set index to be "" and index-prefix to be"$". Here we can see that the entire index row is the same as the previous question but rendered without the prefix because we specified the prefix to be an empty string.

Part 3 - Address Feature

In this scenario, we set the index to be 101, which is 5 in decimal and we specify that the index base is binary. By specifying that the index base is binary, the prefix is automatically set to 0b and the index values are rendered with a binary prefix. In this case as well, we pass in a single index value and it is incremented by 1 appropriately in the mentioned base. Fill in each value as twice the decimal equivalent of the index. For example, at 0101 (5), enter "10"; at 0110 (6), enter "12", etc.
